# 申請ツール 仕様書

## 1. 適用範囲

本仕様書は、申請フォーム機能、管理ダッシュボード機能、マスタ取り込み機能の実装仕様を定義する。

## 2. 用語

- 人事マスタ: 社員情報を保持するマスタ
- 納品先マスタ: 送付先情報を保持するマスタ
- 取込: ExcelファイルをDBへ反映する処理

## 3. 機能仕様

## 3.1 申請登録

### 入力

- 申請種別、機器カテゴリ、共通項目、分岐項目

### 共通項目

- 氏名
- 社員番号
- 会社名
- 部署名
- 役職

### バリデーション

- 氏名: 必須
- 社員番号: 貸与時必須
- 会社名: 必須
- 部署名: 必須
- 役職: 必須
- 送付先: 貸与時必須

### 処理

1. クライアント側バリデーション
2. `POST /api/applications` 実行
3. サーバー側で再バリデーション
4. `Application` レコード作成
5. 受付通知メール送信（失敗しても申請保存は成功扱い）

### 出力

- 成功: `201` + `id`, `status`
- 失敗: `400`（`fieldErrors`）または `500`

## 3.2 氏名・社員番号補完

### 仕様

- 氏名入力時に人事マスタ候補を表示
- 社員番号はプルダウン候補から選択可能
- 氏名 + 社員番号一致時に、会社名・部署名・役職を自動入力

### 補足

- マスタ不一致時は自動補完せず、手動入力で継続可能

## 3.3 納品先入力

### 入力モード

- 選択入力
- 手動入力

### 選択入力

1. 会社名を選択
2. 会社名に紐づく拠点を選択
3. 住所を自動入力

### 手動入力

- 住所を直接入力可能
- 送信時に納品先マスタ照合
- 未登録の場合はエラーで登録不可

### エラーメッセージ

`入力された送付先は納品先マスタに登録されていません。会社名/拠点を選択するか、管理者にマスタ登録を依頼してください。`

## 3.4 マスタ取込

### 対象

- `employees`（人事マスタ）
- `destinations`（納品先マスタ）

### API

- `POST /api/admin/imports/[masterType]`

### 取り込みルール

- Excelは1シート目のみ読み込み
- ヘッダ揺れは許容見出しで吸収
- 必須列欠落行はエラー
- 主キー重複時はアップサート

### 結果

- `totalRows`, `successRows`, `failedRows`, `errors[]` を返却

## 4. マスタ項目仕様

## 4.1 人事マスタ

- 主キー: `employeeCode`
- 必須: `employeeCode`, `name`（取込時に `positionCode` も必須）
- 任意: `kanaName`, `departmentCode`, `departmentName`, `positionName`, `employeeType`, `email`
- 管理項目: `normalizedName`, `normalizedDept`, `isActive`, `sourceFileName`, `importedAt`

## 4.2 納品先マスタ

- 主キー: `searchKey`
- 必須: `searchKey`
- 任意: `companyName`, `siteName`, `postalCode`, `address`, `buildingName`, `phone`
- 管理項目: `normalizedName`, `isActive`, `sourceFileName`, `importedAt`

## 5. 画面仕様（抜粋）

## 5.1 管理画面

- `/admin/imports`: Excel取込実行、結果表示
- `/admin/masters/employees`: 人事マスタ表示
- `/admin/masters/destinations`: 納品先マスタ表示

## 5.2 申請画面

- 共通項目入力
- 分岐項目入力
- 確認ダイアログ
- 登録完了通知

## 6. エラー仕様

- バリデーションエラー: `400` + `fieldErrors`
- 業務エラー（手動住所未登録等）: `400`
- システムエラー: `500` + `internal_server_error`

## 7. 制約事項

- 現状の社員マスタ項目に会社名が明示的にないため、自動補完の会社名はデータ整備状況に依存する
- 住所照合は登録済み納品先マスタを前提とする
