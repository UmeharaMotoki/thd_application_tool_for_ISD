## 2. AWSサービス一覧

### 2.1 コンピューティング

| サービス | 用途 | 詳細 |
|---------|------|------|
| **ECS（Fargate）** | アプリケーション実行基盤 | Next.js（App Router）コンテナをホスト。ローリングアップデートによるゼロダウンタイムデプロイ |
| **EventBridge** | バッチスケジューリング | マスタデータ取り込みバッチを毎朝4:00にトリガー。下書き自動削除バッチも管理 |
| **Lambda** | バッチ処理（TBD） | ECSタスクの代替として検討中。マスタ取り込みバッチの実行 |

### 2.2 ネットワーク・配信

| サービス | 用途 | 詳細 |
|---------|------|------|
| **CloudFront** | CDN・HTTPSエンドポイント | 静的アセットのキャッシュ配信。HTTPS強制 |
| **ALB（Application Load Balancer）** | ロードバランサー | ECSコンテナへのトラフィック分散。ヘルスチェック対応 |

### 2.3 データストア

| サービス | 用途 | 詳細 |
|---------|------|------|
| **RDS（PostgreSQL）** | メインデータベース | 申請データ・マスタデータ・下書き・監査ログを保管。7テーブル構成 |
| **S3** | ファイルストレージ | 添付ファイル（Presigned URL方式）・DBバックアップ・マスタExcelファイルを格納 |

### 2.4 セキュリティ・設定管理

| サービス | 用途 | 詳細 |
|---------|------|------|
| **Systems Manager Parameter Store** | 設定値管理 | DATABASE_URL・AZURE_AD_CLIENT_ID・SALESFORCE_INSTANCE_URL等の環境変数を管理 |
| **Secrets Manager** | シークレット管理 | NEXTAUTH_SECRET・AZURE_AD_CLIENT_SECRET・SALESFORCE_CLIENT_SECRET・ITAM_API_KEY等の機密情報を管理 |

### 2.5 監視・運用

| サービス | 用途 | 詳細 |
|---------|------|------|
| **CloudWatch** | ログ・監視・アラート | エラー率・応答速度・CPU使用率の監視。閾値超過時に管理者メール通知。5分間隔でヘルスチェック |

### AWSサービス全体マップ

```
[CloudFront] → [ALB] → [ECS（Next.js）]
                              ├── [RDS PostgreSQL]（申請・マスタ・下書きデータ）
                              ├── [S3]（添付ファイル / バックアップ / マスタExcel）
                              ├── [CloudWatch]（ログ / 監視 / アラート）
                              └── [Parameter Store / Secrets Manager]（環境変数 / シークレット）

[EventBridge] ──→ [ECS Task / Lambda]（バッチ：毎朝4:00）
                        └── [S3]（マスタExcel読み込み）
                        └── [RDS]（DB更新）
```

## 3. Azureサービス一覧

### 3.1 Microsoft Entra ID（旧 Azure AD）

| サービス | 用途 | 詳細 |
|---------|------|------|
| **Microsoft Entra ID** | 認証基盤（IdP） | OIDC（OpenID Connect）リダイレクト方式でシングルサインオンを実現 |
| **条件付きアクセスポリシー** | アクセス制限 | 社内VPN外からのアクセスを自動ブロック |

#### Entra ID認証フロー

```
[ユーザー] → /login
     │
     ▼
[NextAuth.js（Auth.js）Azure ADプロバイダー]
     │
     ▼ OIDCリダイレクト
[Microsoft Entra ID]
     │ 認証成功
     ▼
[セッション発行]（HttpOnly + Secure Cookie、有効期間8時間）
     │
     ▼
[アプリケーション利用]
```

### 3.2 Microsoft 365

| サービス | 用途 | 詳細 |
|---------|------|------|
| **Microsoft Graph API** | メール送信 | 申請受付通知・障害アラート・バックアップ失敗通知等のメール送信。SMTPフォールバックあり |

#### メール送信対象

| タイミング | 送信先 |
|-----------|--------|
| 申請完了時 | 申請者・情報システム部 |
| SF連携失敗時 | 管理者 |
| ITAMフォールバック発生時 | 管理者 |
| バックアップ失敗時 | 管理者 |
| マスタ取り込み失敗時 | 管理者 |
| 異常検知時 | 管理者 |
| APIキー期限1ヶ月前 | 管理者 |
| SSL証明書期限1ヶ月前 | 管理者 |

## 4. 外部システム連携一覧

| システム | 連携方向 | タイミング | タイムアウト | リトライ |
|---------|---------|-----------|------------|---------|
| **Microsoft Entra ID** | 外部→本システム | ログイン時 | Entra ID側SLAに依存 | なし（Entra ID側に依存） |
| **Microsoft Graph API** | 本システム→外部 | 各通知イベント発生時 | - | 最大5回（5分間隔） |
| **Salesforce REST API** | 本システム→外部 | 申請完了時（書き込みのみ） | 10秒 | 最大5回（指数バックオフ：1・5・15・30・60分） |
| **ITAM API** | 双方向 | フォーム入力時（読み取り）・申請完了時（書き込み） | 5秒 | 1回のみ |

## 6. データフロー

### 申請フロー

```
[ユーザー]
  │ 申請フォーム入力
  ▼
[ECS：Next.js]
  ├── ITAM API（資産番号入力時に資産情報取得）
  ├── RDS（社員マスタ・納品先マスタ照合）
  │
  │ 送信
  ▼
[ECS：Next.js]
  ├── RDS（applicationレコード作成）
  ├── S3（添付ファイル保存 via Presigned URL）
  ├── Microsoft Graph API（受付通知メール送信）
  ├── Salesforce REST API（申請データ書き込み）
  └── ITAM API（資産管理情報書き込み）
```

### マスタ取り込みバッチフロー

```
[EventBridge（毎朝4:00）]
  │
  ▼
[ECS Task / Lambda]
  ├── S3（マスタExcelファイル取得）
  ├── RDS（employees / delivery_locations テーブルをアップサート）
  └── RDS（master_import_logsに結果記録）
       └── 失敗時: Microsoft Graph API（管理者へメール通知）
```

### 認証フロー

```
[ユーザー] → [CloudFront / ALB] → [ECS：Next.js Middleware]
                                        │ セッション未認証
                                        ▼
                                   [Microsoft Entra ID（OIDC）]
                                        │ 認証成功
                                        ▼
                                   [セッション発行（HttpOnly Cookie）]
                                        │
                                        ▼
                                   [アクセス制御（ロールチェック）]
                                    ├── 申請者 → /apply/*
                                    └── 管理者 → /admin/*
```

## 7. セキュリティ構成

| レイヤー | 施策 |
|---------|------|
| **ネットワーク** | 社内VPN経由のみアクセス可（Entra ID 条件付きアクセス） |
| **通信** | HTTPS強制、CSP / X-Frame-Options / HSTS等のセキュリティヘッダー |
| **認証** | Microsoft Entra ID OIDC、セッション有効期間8時間、30分無操作でタイムアウト警告 |
| **認可** | Next.js Middleware でロールチェック（申請者 / 管理者）、IDOR対策 |
| **シークレット** | AWS Secrets Manager（秘密鍵）・Parameter Store（設定値）で管理 |
| **DB** | Prisma ORM によるプレースホルダー（SQLインジェクション防止）、生クエリ禁止 |
| **ファイル** | S3 Presigned URL（一時アクセスのみ）、サーバー側でMIME・拡張子・サイズ検証 |
| **レートリミット** | IPベース 1分間30リクエスト上限、超過時429 |
