## 1. PC貸与

### 1-1. 一次受付フロー

**前提条件：貸与保留がないこと**

#### ステップ

1. **ITAM検索・重複確認**
   - 対象者の社員番号でITAMを検索
   - 本人確認・同用途での二重貸与がないことを確認

2. **申請内容の確認（用途の充足チェック）**
   - 記載された利用用途が充足しているか確認
   - 不足がある場合は差し戻し（以下の項目を追記依頼）

3. **仕様パターンの確認**
   - 社内仕様 / 社外仕様 / 工場出荷 を確認・判断

4. **貸与保留チェック**（→ 6章「貸与判断基準」参照）

5. **担当者へToDoを送付**（キッティング依頼）

---

### 1-2. 申請差し戻し条件と対応文例

| 差し戻し事由 | 内容 |
|---|---|
| 用途不明（社内仕様） | 客先・契約期間・機器利用場所・利用用途詳細・管理者権限の有無と理由を追記依頼 |
| 用途不明（社外仕様） | 上記に加え、社外仕様必須理由・CYNCSプロジェクト番号・Office見積ケース番号を追記依頼 |
| 仕様パターン未選択 | 社内仕様／社外仕様の違いを説明し再申請依頼 |
| 故障交換申請の誤申請 | ヘルプデスク窓口へ誘導 |
| 複数台を個別申請 | 1申請にまとめるよう案内 |
| 機器貸与延長申請 | 延長申請の正しい方法をFAQで案内 |
| 派遣先要請（覚書未締結） | 覚書締結・CSR推進部景山次長の承認を取得後に再申請依頼 |

---

### 1-3. 仕様パターン別チェックポイント

#### 社内仕様
- 社内ネットワーク・共有フォルダへの接続可
- 客先ネットワークへの接続不可

#### 社外仕様
- 客先ネットワークへの接続可
- 社内ネットワーク・共有フォルダへの接続不可
- Office はライセンス別途購入 → 情報システム部でインストール後出荷
- ウイルスバスターはインストール済み
- AFE利用の確認は砂田さんへToDo

#### 工場出荷
- OSのみインストール、Officeなし
- PCの設定・ソフトウェアのインストールは関知しない
- ウイルスソフト：Microsoft Defender インストール済み
- 返却時は購買へ直接連絡（横河レンタ・リースは個別契約）

## 2. PC返却

### 2-1. 返却受付フロー

1. **開梱・基本確認**
   - 箱とPC本体の管理番号を照合

2. **契約年数確認（MyRentecで確認）**

#### 契約年数残2ヶ月以上の場合
3. PCを起動し、OS起動前にF12キーを連打
4. [BIOS SETUP] → [General] → [Battery] でバッテリー状態を確認
   - **Excellent のみOK**
   - それ以外 → 玉突き交換を実施

5. 養生テープを左上に貼り必要項目を記載、所定の場所へ収納

#### 当月解約予定の場合
3. バッテリー状態確認はなし
4. 養生テープを左上に貼り必要項目を記載、所定の場所へ収納

---

### 2-2. 破損確認と対応

| 破損状況 | 対応 |
|---|---|
| バッテリー膨張による筐体割れ | 玉突き交換を実施 |
| 画面のノイズ | 玉突き交換を実施 |
| 3つ以下のボタンの取れ | 玉突き交換を実施 |
| 原因不明のPC起動不可 | 玉突き交換を実施 |
| 落下・水こぼしおよびそれらが原因と思われる破損 | リスクマネジメント報告が必要 |
| 4つ以上のボタンの取れ | リスクマネジメント報告が必要 |

**リスクマネジメント報告が必要な場合：**
- 起案済みであれば玉突き交換を実施
- 未起案の場合は申請のレビューにて破損写真とともに申請起案者へ報告

---

### 2-3. 玉突き交換の実施手順

1. オリックス・レンテックへメールで玉突き交換を依頼
   - 宛先：`supportdesk_ict@orix.jp`
   - CC：`sales2.2_tokyo@orix.jp`、`ryo.horibuchi.wh@orix.jp`、`thd_orix.365ml@orix.jp`、`apac-it-asset@technopro.com`

2. メールに記載する必須情報：
   - OIR番号
   - モデル名
   - 故障概要
   - 故障PC回収希望日・回収場所
   - 代替PC納品希望日・送付先

3. 返却（格納）
   - 養生テープを左上に貼り、必要項目を記載して所定の場所へ収納

## 3. スマホ・通信機器貸与

### 3-1. 一次受付フロー（通信機器）

**前提条件：貸与保留がないこと**

1. **ITAM検索・重複確認**
   - 対象者の社員番号でITAMを検索
   - 本人確認・同用途での二重貸与がないことを確認

2. **申請内容の確認（用途の充足チェック）**
   - 不足がある場合は差し戻し（追記依頼）

3. **派遣先要請の場合の追加確認**
   - TCS以外：覚書の締結 + CSR推進部景山次長の承認が必須
   - TCS：覚書不要（TCS本社がCSRに年間一括承認済み）・レビュー確認のみでOK

4. **担当者へToDoを送付**

---

### 3-2. スマホの仕様区分と対象者

| 仕様 | 主な対象者 | 特徴 |
|---|---|---|
| 通常仕様スマホ | 管理社員（営業職等）、技術社員リーダー業務用 | 通話・メールアプリ・SMS・テザリング、個人設定付与（社員番号紐づけ）、カメラ無効化 |
| 一時貸与スマホ | 営業事務・派遣事務の在宅勤務用、派遣契約分、在宅勤務用 | 通話・OWA・SMS・テザリング、メールはOWAで確認、カメラ有効化、社員番号不要 |
| ガラホ | 通話のみ利用、カメラ持ち込み禁止の客先 | 通話・SMS利用可、テザリング機能なし（オプション設定で2GB/月可） |
| 通信カード | データ通信のみ | HUAWEI HWD12（制限解除）、通信量7GB/月 |

---

### 3-3. 申請差し戻し条件

| 差し戻し事由 | 対応 |
|---|---|
| 通常仕様申請で社員番号がない | 社員番号が分かり次第、用途/理由欄に追記のうえ再申請依頼 |
| 用途/理由欄が不明確 | 利用用途の詳細を追記依頼 |
| 技術社員用で客先・契約期間・機器使用場所の記載なし | 3項目の追記依頼 |
| 派遣先要請で覚書未締結・CSR承認なし | 覚書締結とCSR推進部景山次長の承認取得を依頼 |

---

### 3-4. 技術社員用：仕様判断フロー

```
技術社員用スマホ申請
├── リーダー業務用 → 通常仕様スマホ
├── 受託・構内請負
│   ├── メールアプリ利用あり → 通常仕様スマホ
│   └── メールアプリ利用なし → 一時貸与スマホ
└── 派遣契約・在宅勤務用 → 一時貸与スマホ
```

### 3-5. 返却期限の設定ルール

返却期限を設定するもの：
1. 機器利用場所が客先（最大1年間）
2. 派遣先要請貸与機器（貸与期間＝契約期間、延長見込みがあるものは最大1年）
3. 研修用で研修期間が決まっているもの
4. 出張期間のみ利用で出張期間が決まっているもの（出張兼用は貸与期間なし）
5. その他、この期間だけ利用という申告があるもの
6. 棚卸の必要があり返却予定日をトリガーにするもの

**リーダー業務用は原則返却期限なし**（業務内容から拠点・自宅・客先のどこでも利用するため）

### 3-6. スマホ端末設定手順（一時貸与スマホ）

**必要なもの：** ベースキッティング後の端末、割賦無しのSIM、テプラシール（テレワークA～）、SIMピン、SIM台帳

**設定手順：**
1. SIM台帳で使用するSIMのセルの色を青→赤に変える
2. SIMを挿入する
3. SIM番号のシールを箱に貼付し、[製造番号]欄に端末のSP番号を記入
4. 電源ON → パスコード「147258（6桁）」を入力
5. テプラシール（テレワークA～）を端末と箱に貼付
6. Safariを開き、履歴を消す
7. 設定 → 一般 → 情報 → 「名前」が「SP番号」になっているか確認
8. ソフトウェアアップデートがあれば実行
9. 設定 → バッテリー → バッテリー残量(%) → 有効化（緑色）
10. 設定 → App Store → 自動アップデート・自動ダウンロード・常に許可を有効化
11. 設定 → 電話 → [自分の番号]がSIMの番号と一致しているか確認
12. 設定 → Hub → [位置情報] → [常に] に設定
13. アップデート完了後、HubアプリでデータシェアとアプリロックをON
14. 充電を60%以上にして電源OFF
15. スタートアップガイドを同梱し、輪ゴムでくくる

---

### 3-7. スマホ端末設定手順（通常仕様スマホ）

**設定後の検品手順：**
1. SIM番号シールの[製造番号]欄に端末のSP番号を記入
2. 電源ON → パスコード「147258（6桁）」を入力
3. Safariを開き、履歴を消す
4. 設定 → 一般 → 情報 → 「名前」が「社員番号」になっているか確認
5. 設定 → メール → アカウント → TechnoProのアカウントに利用者のメールアドレスが正しく反映されているか確認
6. 設定 → 電話 → [自分の番号]がSIMの番号と一致しているか確認
7. 充電を60%以上にして電源OFF

- 一括清算（請求先：ピーシーアシスト）での発注を行う

### 3-9. 注意事項

- **TCSのガラホ新規貸与：** 事前にTCS→景山さんへ確認済みのメールを添付してもらい、貸与保留案件確認後そのまま町田さんへToDo送付

- **スマホの個人情報：** 利用者変更・共有利用は個人情報保護法の指針によりコンプライアンス上不可
- **客先派遣用のスマホ：** 一時貸与スマホ一択
- **レンタル品の転貸借：** テクノプロとは別の会社への貸与には転貸借契約が必要（基本的に不可）

## 4. スマホ・通信機器返却

### 4-1. スマホ返却受付時の作業フロー

1. **基本確認**
   - 箱のSP番号と端末のSP番号が同一か確認
   - ワークフローを確認（なぜ返却されたのかを把握）
   - ITAMを確認し、資産状態が「返却」になっているか確認

2. **電話番号の確認**
   - 端末の電源を入れて電話番号が合っているか確認
   - パスコードが解除できない場合：外線を使用して着信すれば確認可能

3. **LCM対象確認**
   - ITAMで返却された番号を検索
   - LCM用のスマホが登録されている場合 → 回線切り替えを実施 → 切り替え後の端末は遊休在庫に → 返却されたスマホのSIMは廃棄

4. **割賦の有無確認**
   - 割賦確認のエクセルファイルで確認

#### 割賦ありの場合
- 本体からSIMを抜き、SIM台帳にセロテープで貼る
- SIM台帳に電話番号を記載
- 本体に付箋を貼りDatawipeを依頼

#### 割賦なしの場合
- 本体に付箋を貼り、抜いたSIMを付箋に貼り付けてDatawipeを依頼
- 解約するスマホの電話番号をオーダーフォームに記載
- 町田さんにKDDIへの解約を依頼

### 4-2. スマホ返却後のITAM更新

#### 資産側の更新

**新規登録側（新端末）：**
- 資産状態 → [改番＆新規] または [解約]
- 用途 → [販売管理費（THD在庫）]
- 返却日 → 処理実施日
- チケット番号 → 対応したITサービス依頼番号

**返却端末側：**
- 資産状態 → 以下の区分で更新
  - iPhone SE → [廃棄]
  - iPhone 7 → [修理対応中]
  - iPhone SE2,3（正常） → [遊休]
  - iPhone SE2,3（故障） → [修理対応中]
- 貸出日・返却予定日 → 削除
- 返却日 → 処理実施日

#### リレーション側の更新

**新規登録側：** 管理部署No・人事No を返却端末側からコピー  
**返却端末側：** 管理部署No → 060000547、人事No → 0000000000

### 4-3. LCM対象者の回線切り替え手順

1. 返却されたスマホとLCM切り替え後のスマホを用意（電話番号の一致を確認）
2. `www.kddi.com/kirikae/` にアクセス
3. 「iPhone・iPadに変更される方」をクリック
4. 「データ移行済み」と「内容確認済み」にチェック
5. 必要事項を入力：
   - 申し込み区分：機種変更
   - au携帯電話番号：回線切り替えを実施するスマホの電話番号
   - 申込書番号：切り替え後端末の箱側面シールに記載
   - 契約名義人：テクノプロ・ホールディングス株式会社
6. 回線切替完了後、スマホ2台の電源を切ったまま15分待機
7. 15分後、切り替え後の端末の電源を入れる
8. 電話番号（0120-929-494）へ電話し、発信テスト実施
9. 外線から端末の電話番号に発信し、切り替え後の端末で受信できるか確認

## 6. 貸与判断基準（共通）

### 6-1. 貸与保留の判断フロー

**対象：** PC貸与申請・通信機器貸与申請のみ  
**例外：** 入替・故障交換・LCMなどは貸与判断ステップを介さず手配を進める

#### ステップ

1. **申請を確認したら、対象機器の経費負担部署でITAMを検索**
   - 注意：所属は○○支店でも別部署での業務の場合は、その部署コードで検索

2. **以下3点を確認**

| 確認項目 | 内容 |
|---|---|
| ①返却期限超過機器 | 返却期限が超過している機器 |
| ②退職・休職者の運用中機器 | 退職・休職しているのに「運用中」となっている機器（休職は1か月以内に復職予定なら見逃す） |
| ③低利用機器 | 最新インベントリが2か月以上前の機器 |

3. **いずれかに該当 → 貸与保留**
   - レビューで部署担当者に連絡（テンプレートあり）
   - 自分宛にToDoを送付して管理
   - 保留解消まで手配は実施しない

4. **保留中に別申請が来た場合**
   - 先行申請での保留中の旨をレビューで伝える

5. **貸与保留のみは差し戻しをしない**
   - ただし、用途不足かつ貸与保留がある場合は差し戻しと保留の両方を伝える
